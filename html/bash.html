Interactive Shell

Type exit or use Ctrl-D (end-of-file) to terminate a session. If it is a login session you can also type the logout command.

A command line is a line of text ending in \n. The first word of a command line is the command and the rest are arguments (also called parameters). An option is an argument that controls some behavior of the command. Options usually take the form of a single hyphen followed by a single letter or a double hyphen followed by a word. Options can take arguments of their own.

Snippets

  • PS1=\$\  sets the prompt to a simple '$' character. There needs to be a space entered at the end after the last '\'. It's useful to generate command line session transcripts for documentation pages with hilite.me.

Notes

  • Using printf instead of echo to write strings from the command line to a file makes actual newlines appear in the file instead of literally "\n", which is not what we want.

chmod nnn file, chmod [ugoa] [+|-] [rwx] Changes the permissions associated with a file. The 3 digits in the first argument of the first version represent permissions for the user, group & everyone else, in that order. Let r=4, w=2, x=1. Then add the values that represent the permissions you wish to assign for each digit & add them up.

In the second version the first set of characters represents the category of users the change will apply to, where u=user, g=group, o=others, & a=all. + adds the given permissions & - removes them. r=read, w=write & x=execute.

The -R option runs the command recursively for a directory & its subdirectories.
umask Displays the umask value which, by default, masks the permission values of 666 for a new file & 777 for a new directory.
